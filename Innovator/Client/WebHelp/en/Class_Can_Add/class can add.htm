Can Add Permissions for Class Structure

Introduction

Class Structure allows a single Aras Innovator ItemType to be specialized into classes and sub-classes that can be used to organize properties, launch different input forms, and also set permissions for instances of an ItemType using Can Add settings.

For example, an ItemType called Inspection Result might be classified into Aesthetic, Packaging, Tolerance, Product and Software classes.  Using Classification different groups and identities can use the same Inspection Result item in different ways based on business requirements.

In this help page we learn how to grant specific groups/identities the rights to create (i.e. Can Add) the Inspection Result item.  For details on using class-based properties and class-based forms see the related Class Specific Properties and Class Structure help pages.

Defining Classification Specific Create Permissions for an ItemType

To explain how Classification Specific Create/Can Add permissions work in Aras Innovator we’ll use a theoretical ItemType called Inspection Result.  Assuming the classifications Aesthetic, Packaging, Tolerance, Product and Software classes have already been added, this is what we would see in its Classification browser.

 

Adding Permissions to Create Items With No Class/Type Restrictions

Creation rights are assigned under the Can Add tab on the itemtype window.  Here is where we specify who can add (i.e. create) instances of this ItemType.  Prior to any assignments in a brand new ItemType, no groups or individual identities can create an Inspection Result because Can Add will be empty.

Below the Inspectors group identity has been added to the Can Add grid to allow members of the Inspectors group to create records in the database.  By leaving the classification column (far right) blank group members can create any Class type of inspection results.  In other words we have not restricted the Inspectors to any one classification of Inspection Results:


Restricting Groups/Identities to Specific Classification Types

In our use case, there are two additional groups involved in inspections - Aesthetics and Packaging.  We need to allow these groups to create Inspection Results but only allow them to create items of the appropriate Class Type.  This is done by specifying the Class Path in the Can Add row.

Here, three additional groups have been added to the Can Add tab and the correct classification has been assigned to each row:


 

Once saved, members of these groups will only be able to create (or select) the configured Class Path Types.  Make sure you check the Can Add check box.  By specifying the Class Path, these groups will only be able to work with the sub-types specified.  Changes are committed when you save the ItemType.

Once again please keep in mind that if no classifications are specified that Group/Identity will be able to create results of any type. 

How Classification Specific ‘Can Add’ Rights are Enforced

When items (i.e. Inspection Results) are created, the user creating the Inspection Result will be given only the eligible choices in the Classification selection window.  In this way, each group ‘Can Add’ only the classifications that the administrator configured for the business rules in place.
