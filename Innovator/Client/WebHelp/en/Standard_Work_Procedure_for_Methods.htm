Standard Work Procedure for Methods

  1. Gather documents defining Items you will be working with
    1. Item/Relationship diagram with ItemType names
    2. List of property names and data_types for each Item
  2. Make a plan
    1. Client or Server
    2. How to call
    3. What input
    4. Desired result
  3. Walk through process in UI
    1. Make sure that Innovator will do what you want
    2. Consider effect of lock status on your method
    3. Consider effect of missing or invalid data on your method
  4. Pseudocode
  5. Code
    1. Use a variable naming convention
    2. Test any XPath
    3. Test any AML
    4. Syntax check
  6. Set up calling method
  7. Test
