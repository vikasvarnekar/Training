Type-Classification

ItemTypes may contain a structure of subclasses where each class can refer to its own specific properties, forms and lifecycle. Classification allows you to share properties among subclasses. If you define a property at a parent level class, all subclasses inherit those properties. Each of these subclasses can have additional specific properties in addition to the properties defined by the parent ItemType. A class structure can be many levels deep and wide depending on the complexity required.

For example, in the CAD Form, click to view the class structure for CAD Item Type.

The classification dialog box is displayed.

 

In the example, the CAD ItemType has a defined class structure.  A CAD can be represented as Mechanical or Electronic.  Because each of the subclasses share common attributes it can be more efficient to use class structure versus creating entirely new ItemTypes.
