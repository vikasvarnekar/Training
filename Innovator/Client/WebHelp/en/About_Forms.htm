About Forms

A Form is a dynamically generated page that contains ItemType property information and acts as an area for data entry for end-users.  Forms are designed by administrators to present relevant information in a predetermined layout and can be the basis for end-user interaction through the use of controls (checkboxes, buttons, menus, etc.), and labels on those controls.  Users generally 'complete' a Form by modifying the control value (entering text, selecting menu items, etc.), before submitting (saving) the Form.

Let us consider an example of the Part ItemType to know the various components of a Form.

Part Item Window

Item Menu:  The Item menu is similar to the main menu and you can perform various generic tasks such as creating a new Item, saving the current Item, claiming/unclaiming the current Item, and so on.  For more information, refer to Item Menu.

Item Toolbar:  The Item toolbar displays the list of buttons for performing common tasks such as add, view edit, save, copy and save.  For a description of the buttons on the Item toolbar, refer to Item Toolbar.

Item Information:  The Item information displays the ItemType and the image associated with the ItemType.

Form Fields:  The Form fields are the area for data entry for the users.  The administrator designs the form and determines the fields to be inserted in a form for an ItemType.  Thus, depending on the ItemType and the administrator configuration for the ItemType, the fields in a form may vary.  For more information on different field types, refer to Form Field Types.

Tabbed View:  The tabbed view  comprises the various tabs associated with the Item.  You can click on each tab to view and access the tab contents.

Relationship Toolbar:  The relationship toolbar provides buttons to perform various tasks such as adding a related Item, replacing a related Item, promoting the Related Item, and so on.  For information about the various buttons in the Relationship toolbar, refer to Relationship Toolbar.

Relationship Grid:  The relationship grid lists the related/child Items.  The grid comprises of Relationship Item properties and child Item properties.  The highlighted properties belongs to the Relationship Item, while the non-highlighted properties belong to the child Item.
