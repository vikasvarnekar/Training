Creating Relationship

A Relationship binds two Items - the source (parent) Item and the related (child) Item, making it possible for the source (parent) Item to reference the information held in the related Item.  The relationship is created by starting at the source (parent) Item, and creating a link to the related (child) Item.

Depending on the ItemType configuration done by the administrator, either or all of the options to add a related Item are enabled and available.  The three main relationship types are as follows:

Creating a Relationship with Related Item (Pick Related or Create Related)

When you add a relationship, if there are only two options:Pick Related or Create Related, it is mandatory for you to add a related Item. You can either pick an existing Item ( Pick Related) or create a new Item (Create Related) to be added, but associating a related Item is mandatory in such cases.

Example: Part-BOM relationship. For detailed steps, refer to Creating a Relationship with Related Item.

Creating a Relationship with related Item being optional (Pick Related, Create Related , or No Related)

When you add a relationship, if there are all the three options: Pick Related, Create Related, or No Related , it is optional for you to add a related Item. You can either add a related Item (Pick Related or Create Related) or opt to not associate any related Item ( No Related) and add only information about the relationship.

Example: Part-Alternate relationship. For detailed steps, refer to Creating a Relationship with Optional Related Item.

 Creating a Relationship that cannot add a related Item (No Related)

When you add a relationship, if there is only one option: No Related, you cannot associate any related Item (No Related), but can only add information to the relationship.

Example: List-Value relationship. For detailed steps, refer to Creating a Relationship that cannot add a Related Item.
