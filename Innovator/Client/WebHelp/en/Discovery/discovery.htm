About Discovery Privilege

Discovery Privilege provides the capability to further restrict access to items in your Aras Innovator database.  Discovery privilege determines whether identities are allowed to learn that an item exists in the Aras Innovator database.  This feature can be used to restrict a development or supply chain partner to only discover the items in your Aras Innovator database that they are responsible for.  In this case, only items for which “Can Discover” access is set will be returned in searches even if other items match the search criteria. 

Discover and Get Access Behaviors

 

Access Type

Aras Innovator Behavior

No Get or Discover

(No Access)

Items will not be returned in main grid or relationship grid even if they match the search criteria

Warning will be displayed in the status bar indicating a permission restriction if “Show Permissions Warning” is set and there are items missing from the result due to insufficient privilege

Discover Only Access

Items will be returned in main grid and relationship grid when they match the search criteria

Users will be restricted to viewing only the information returned in the grid and will not have the ability to open the form

A warning will be displayed in the lock status column of the main grid

A warning will be displayed in the status bar indicating a permission restriction if “Show Permissions Warning” is set and there are items missing from the result due to insufficient  privilege                                                                                                                                                                                                    

Get Access

No changes here

Users will be allowed to view all information and open forms

 

Aras Innovator can be configured to alert the user that there are items not shown due to permission restrictions.

Discover and Get Access Scenario

The following scenario demonstrates how to restrict the “All Suppliers” Identity from discovering preliminary parts.  In this example, the “New Part” and “Released Part” permissions have been edited as shown.

 

Results when “Get” Access is defined

Search Screen Results when logged in as a member of the “All Employees” Identity

This Identity has “Get” access to both of these items

No warnings are displayed

 

Results when only “Discover” access is defined

Search Screen Results when logged in as a member of the “All Suppliers” Identity

This Identity has “Discover”  but not “Get” access to one of these items

This Identity has no access to one of these items (it is not displayed in the results)

Discover Only Icon is displayed in the first column

Warnings are displayed in the status bar indicating that there are additional items

 

The primary function of a “Discover” privilege is to allow the user to search for items and view limited information about them, but not view the detailed information that would be present on the item form.  This feature provides a certain level of access control over sensitive information.
