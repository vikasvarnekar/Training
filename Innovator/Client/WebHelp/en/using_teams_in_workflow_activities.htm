Using Teams/Roles in Workflow Activities (Administrators Only)

Support is provided for team identities/roles to be specified as assignments for a workflow map activity.  This allows for dynamic workflow assignments which are based on the group of users (Team) which are responsible for the work that is to be completed.  When a workflow process is instantiated from the workflow map, the team identity or role assignments will be converted into individual assignments for each member of the team or role.  For example, if the Team identity is specified as an assignment on a workflow map activity for a particular item, then the resulting workflow process created from the workflow map will have individual assignments for each member of the team.  Different rules apply based on the assignment attributes defined in the workflow map (see examples, below).  Similarly, if a Team Role (i.e. Team Manager) is specified as an assignment for a workflow map activity for a particular item, then the resulting workflow process created from the workflow map will have individual assignments for each member on the team with that role (see examples, below).

A team item must be set for the team_id property on each item instance to which the workflow applies.  This provides the necessary team information for the system to utilize for activity assignments in the workflow process.

Steps:

  1. While viewing an activity within an existing workflow map (edit mode), click the Add New Relationship button on the Assignments relationship toolbar.  The System will launch a search dialog populated with the existing identities in the system.
  2. Using the blue bar search, filter for identities with Is Alias=0 and Classification=Team.  This will return the available system team role identities which can be used directly in the activity assignments.

 

  1. Select one or more identities from the search dialog and click . The System populates the selected team identities (roles) in the relationship grid.  These special team identities can be used in combination with other identities in defining the Assignments for the workflow activity.
  1. Set the remaining Activities on the Activity tab.

  1. Click the Process Variables tab and add the appropriate Workflow Map Variables.
