Logon Users

To access Innovator, each user is typically asked to log in through the Innovator authentication screen which requests a user name and password. The user name or login name and password define a Logon User.  In Innovator, when a Logon user is created, the user is automatically assigned an associated Identity.   The link between the Logon User and an Innovator Identity is called an Alias.

 

 

Here is how this concept works in Innovator.  First, the administrator must create a Logon User:

 

1. In the Navigation pane select Administration>Users. The following menu appears:

 

 

 

2. Click Search Users  to see which users are already in the system.

3. Click Create New User. The following dialog appears: 

 

      4. Fill in the following properties on the dialog as described:

 

Login Name - the login user name that the user will enter in the authentication screen to access Innovator.

Password - the password the user will enter in the authentication screen to access Innovator.

- retype the password to confirm it.Confirm Password

Warning: Login Name and Password are case sensitive.

Logon Enabled - check this box if the user has logon privileges. Leaving this box blank will disallow the user access to Innovator through the authentication screen.

 

 

     The following dialog properties are optional, and can contain values on as-needed basis:

 

 

First Name - The first name of the user

Last Name - The last name of the user

Company Name - The user's company name

Email - the user's e-mail address. This information is necessary if the users are to receive notification of specific activities and action items assigned to them and forwarded to their In Baskets.

Telephone - the user's telephone number

Mail Stop - the user's mail stop

Fax - the user's fax number

Home Phone - the user's home phone number

Cell - the user's cell phone number

Pager - the user's pager number

Employee # - the user's employee number. This information is often used by companies to run metrics on employee activity.

Manager - the identity of the user's manager. This information is often used for delegation purposes or even for permissions structures.

Working Directory - the directory accessed by the user.

Starting Page - 

Default Vault - the vault that provides secure storage of files.

 

5. From the Navigation pane, select Administration> Identities and click Search   to display all Identities.

Group identities can be assigned specific permissions. Now this Identity can be grouped with others to create group Identities.  Thus, for each Logon User an Identity is automatically created by Innovator.  You should see that an Identity was created automatically for the above user, and this Identity is an Alias (notice that the Is Alias property is checked, or set to True).

Alias Tab

 

 

Another way to see the identities associated with a specific Logon User is to look at the list under the Alias Tab in the User form.

 

 

1. From the Navigation pane, select Administration > Users. Let's open the user we created previously. Notice the Hide Tabs arrow in the main tool bar of the dialog.  If you click the down arrow, you will see the tabbed properties of this user:

 

 

 

 

There are two tabs, Aliases and Read Vaults:

 

 

 The Alias tab lists all the Innovator Identities that are aliased to (or associated with) this Logon User.  You can also manually add or delete entries from this list.

The Read Vaults tab enables you to add a vault to use to store user information.
