Forms for TOC Categories

Starting with Innovator 9.4, Aras supports forms that may be attached to the categories listed in the Table of Contents (TOC). When a user clicks on a category folder in the TOC, the associated form pops up as a view in the main window. See example below:

 

This form has the same capabilities as any other form in Innovator. In order to match a form to a TOC category, Innovator checks for two things:

The name of the form must match the value in the Categories list.

The form must be marked as a “Category Form.”

To see a list of available categories:

Log into Innovator as an administrator.

Go to Administration --> Lists.

Search for and open the list named Categories.

 

The values in the Value column are the legal names for Category Forms. The Form name must be an exact match. This list may be edited to add more categories as required.

To create a new Form for a TOC Category:

Log into Innovator as an administrator.

Go to Administration --> Forms.

Create a new form.

In the Form Properties tab, in the Name field, enter the exact same value as its counterpart in the list above.

Check off Category Form field.

 

Populate the form as required.

Save, unlock, and close .

If you now navigate to the associated category in the TOC and click on the topic folder, the view of the new form should appear in the main window area. See example below.
