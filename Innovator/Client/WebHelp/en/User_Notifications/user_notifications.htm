User Notification Messages

Administrators and other authorized individuals may create pop-up and standard (status bar) notification messages.  Pop-up messages appear in a separate window when received, and when a user first logs on.  Standard messages are indicated in the main Innovator status bar, and the user may choose when to see them by clicking on the message indicator. Then they can read them by clicking on the Notification icon.

 

 

 

Creating a User Notification Message:

As a prerequisite, if you have not already done so, create the itemtype, its classification structure, and any class specific properties.  See Class Structure and Class Specific Properties for more information.

  1. On the TOC, open Administration>Notification Messages
  2. Click Create a New Item.
  3. Enter the desired message and settings:
  • Icon – selects the icon to display next to the message.
  • Use Standard Template – presents a simple message window format, with Title, URL link, Text and OK/Exit button(s) when checked. Otherwise, a custom HTML window can be created.
  • Title – the message title, which appears at the top of the window, and is the title displayed for standard messages in the status bar message listing.
  • Text – the body text of the message window.
  • URL – optional URL which is the link when the Message Title is clicked.
  • Target – the identity to whom the message will be delivered.
  • Expiration Date – the date/time the message will no longer be displayed.
  • Type – Standard or Popup. Standard messages are displayed through the status bar. Popup messages each open in a new window when the messages are polled.
  • Show OK Button and OK Button Text – displays an OK button on the message window, with the button text as entered.
  • Priority – determines the order standard messages are listed in the status bar listing, and the order Popup messages are displayed.
  • Show Exit Button and Exit Button Text – displays an Exit button on the message window, with the button text as entered. When clicked, the exit button closes the Innovator application.
  • Acknowledge – Always or Once. Determines if the message remains in the listing (for standard messages), or pops up every polling interval (for Popup messages). If Once is selected, when the user clicks the OK button, the system stores the acknowledgement, so the message will no longer be displayed.
  • Style Css (for Standard Template messages) – optional formatting for the message window. The following styles are available for modification -

Style Name

Description

.title

Title text

.text

Message text

.notification_ok_btn

OK Button text

.notification_exit_btn

Exit Button text
  • Style Width and Height (for non-Standard Template messages) – specifies the dimensions of the message window.
  • Custom HTML (for non-Standard Template messages) – specifies the entire content of the message window. Substitution for {TITLE}, {TEXT} and {URL} are supported.
  • Save/Unlock/Close the Notification Message.  It will be displayed for the target users at the next polling interval.
  • Messages can be edited and deleted.  Note that messages will not be delivered to any users who already acknowledged it, even if the Acknowledge property for the message is changed to “Always”.  To re-deliver an acknowledged message, delete the message acknowledgement(s) for that message (see below).

Viewing Notification Message Acknowledgements:

An administrator may set the polling interval used by the system to check for new messages, and display pop-up messages.

  1. On the TOC, open Administration>Notification Messages.
  2. Select and view the message for which you want to review the Message Acknowledgements.
  3. Click the toolbar “Hide tabs” button on the tear-off form to deselect it, making the Message Acknowledgements tab visible.
  4. Users who clicked the “OK” button will be listed on the tab.

The messages will not be re-delivered to any users who already acknowledged it.  To re-deliver an acknowledged message, delete the message acknowledgement(s) for that message.

Modifying the Notification Message Polling Interval:

An administrator may set the polling interval used by the system to check for new messages, and display pop-up messages.

  1. Edit On the TOC, open Administration>Variables
  2. Edit the Message_Check_Interval. The default value of 900000 (in milliseconds) is 15 minutes.
  3. Updates will take affect when a user next logs in.
