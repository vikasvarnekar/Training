Search Toolbar

The following is an example of the ItemType Search toolbar. Each Search Toolbar in Aras Innovator uses the same buttons.

 

The following table describes the buttons in the search toolbar.

Button Name

Button

Description

Search

Executes the search.

Clear Search Criteria

Removes the applied search criteria.

Search Modes drop-down

Allows you to select the search mode.  The different search modes available are Hidden, Simple, Advanced, and AML.  For more information on each search mode, refer to Search Modes.

Saved Search

Allows you to select the saved search criteria.

 Saved Search drop-down is displayed only if you have saved a search criteria in the database.  For more information, refer to Saved Searches.

Add Criteria

Provides an additional row to specify search conditions and criteria for Advanced Search.  The Add Criteria button is available only for Advanced Search.  For more information, refer to Advanced Search.

Results per Page

Allows you to specify the number of rows to be displayed in one page.  If left blank, it implies to show all at once.

Previous Page

Displays the previous page in the main grid.

Next Page

Displays the next page in the main grid.

Show Total Pages and Results

Displays the current page number as well as the total number of search results. 

 

Maximum Results 

Specifies the number of rows to be returned by the server.  It is useful, if there is a very large number of Items to be returned after a search.
For example, inserting 1000 restricts the search results to the first 1000 rows.

Effectivity Type drop-down

As of (date)

The Effectivity Type drop-down and Calendar are used only for versionable Items and provide further filtering on the search grid.

 

Current:  Displays the latest (highest) generation of an Item that exists as of today.  The calendar is disabled as the date is set to Today by default.  The latest generation is flagged programmatically by is_current Item property.

Latest:  Displays the latest (highest) generation of an Item that exists as of the date selected i n the calendar matching the search criteria provided .  The modified_on date is used for the comparison.  By default, the calendar is set to Today.  The Latest may return a non-current generation of the Item; but, this would be a very unusual case.

Released:  Displays the latest (highest) released generation of an Item that exists as of the date selected in the calendar.  The latest released generation is based on the Released Date.
