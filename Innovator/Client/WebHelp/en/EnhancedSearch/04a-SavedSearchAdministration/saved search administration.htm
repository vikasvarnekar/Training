Saved Search Administration

Saved Searches appear in the Administration Category in the Table of Contents (TOC)

 

The columns that appear in the Main Grid are:

  • Label: The Label of the Saved Search which appears in the SavedSearches section of the toolbar. This column is blank for searches that are auto_saved.

  • ItemType: The name of the ItemType to which the search applies.

  • SearchMode: When a search is first saved this column displays the Search Mode used to create the search. It may be changed by clicking on the appropriate link in the column and editing the Item. Choosing No UI mode hides the search details when you select Saved Search.

  • If the selected Search Mode cannot display the search criteria, (for example if SimpleSearch Mode is used for a search created in AML Mode, but the criteria cannot be represented in SimpleSearch Mode) the simplest mode capable of representing the criteria is used. The mode hierarchy is Simple, Advanced, AML.

  • Criteria: The criteria for all searches are shown in the same format, Aras Markup Language (AML), the fundamental language of Aras Innovator. Administrators familiar with AML may create Saved Searches directly by creating a new Item when Saved Searches are shown in the Main Grid. For more information about AML see the Programmers Guide or API Classes.

  • Location:  This shows where the search will be used:

Relationships Grid

  • Auto Saved: Checkmarks in this column indicate that the searches were automatically saved.
  • Owned_by_id: Displays the identity that created the search.
  • Managed_by_id: Displays the identity that manages the search.
  • Show in TOC: Checkmarks indicate that the saved search appears in the Navigation pane.

Save Searches for the Main Grid using the Main Menu, and for the Relationship Grid using the Item Window Menu.

To create a search for the Search Dialog either:

  • Create a search for Main or Relationships Grid and edit the Saved Search to change the location.

  • Create a Saved Search directly in the Administration tree:

Auto Saved: Searches are automatically saved when you initiate a search, in which case auto_saved is set to 1. When a Search is Saved from a Menu or created from the Administration tree Auto Saved is set to 0.

owned_by_id: is automatically set to the alias Identity of the user who created the search and has get access to the Item. This can be changed by the manager or the Administrator.

managed_by_id: is automatically set to the alias Identity of the user who created the search. The manager and Administrators have full access to the Item
