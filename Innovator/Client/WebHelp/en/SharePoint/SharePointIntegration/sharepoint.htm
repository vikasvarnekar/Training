Innovator Integration to SharePoint

 

Introduction

Innovator provides the ability to access MS Sharepoint documents directly using a standard configurable approach.  A connection to a SharePoint library is configured by the Innovator administrator.  Once this connection is established, Innovator automatically generates references to the SharePoint documents in that library to enable Innovator users to search, download and upload Sharepoint documents directly from within Innovator.

Innovator follows the Sharepoint 'Library' metaphor. SharePoint libraries are linked individually thus allowing different Innovator groups and users to have permissions to different Libraries.  For example you may decide to set up several Sharepoint folders and grant access differently to some Innovator identity/groups than others.

In this help file we discuss:

  1. Creating Connections to SharePoint Libraries
  2.  Automatic Generation of Document ‘Reference” ItemType
  3. Searching, Downloading and/or Uploading SharePoint Documents
  4. Using SharePoint Document Objects in Innovator

Creating a Sharepoint Library Connection

 

The first step is to link a SharePoint Library to Innovator.  Each Innovator/Sharepoint connection is configured using a special Innovator item which resides under the Administration TOC folder "SharePoint Libraries".  As mentioned above, Sharepoint organizes its documents into Document Libraries.  This is the same level used by Innovator to establish connections. 

Select “New SharePoint Library Definition” from the menu.  The various fields are discussed below.

SharePoint Information and Credentials

 

  1. SharePoint Site URL - URL location of your Sharepoint server (e.g. http://yourdomain/Sharepoint )
  2. SharePoint User - the account that you have designated for this library connection

The SharePoint User serves as an agent for the Innovator connection.  The Innovator administrator may choose from only those libraries that this account is authorized to access (see Library Selection below)

  1. SharePoint User Password - password for the user above
  2. SharePoint User Domain - SharePoint may require that the user’s domain is provided

Library Selection

 

The "Refresh List" button queries for all accessible libraries for the SP user specified (above) and populates the pull-down menu just to its right. 

  1. Sharepoint Document Library Name - the Library that contains SharePoint documents available
  2.  ID (read only, system generated)

If the specified SharePoint user (above) only has rights to the Engineering and Manufacturing Libraries, then these will be the only two available in the pulldown for this configuration.  This is entirely up to your SharePoint implementation and Innovator will suggest connections accordingly.

Automatic Generation of ‘Reference’ ItemType

Once you add a SharePoint Library connection, Innovator will create a ‘reference’ ItemType to represent the documents in that library.  This itemtype acts as a reference to the actual document in SharePoint so that the “Download”, “Upload” and basic search grid functions are available in Innovator.

 

The name and label information for this itemtype is entered in the next three fields on the SharePoint Library Definition form.

Generated Document Type Information

 

  1. Name of Generated Document Type - Name of Generated Doc Item (e.g. “Test SP doc”)
  2. Label of generated item - label for this new itemtype (e.g. “Test SP doc”)
  3. Plural label - plural label for itemtype (e.g. “Test SP docs”)

This Innovator itemtype is automatically generated once saved.  It will be named according to your inputs.  This itemtype is used by Innovator to represent each document found in the SharePoint Library you connect to with this SharePoint Library Definition.

You are now ready to check the connection!

Test the Connection

Now you will be able to go to the Innovator TOC and open the SPDocuments Folder.  Within it should be the Generated Document Items you specified above, however Innovator will populate the grid with all documents found in the Library you’re connected to for that reference Document ItemType.

 

If you do not see the SPDocuments Folder in the TOC, it’s possible that you do not have permission to Discover or Get the item.  Check the itemtype definition as you would for any other Innovator itemtype.

If you need additional customization to meet your specific business goals, Innovator’s open framework can support further refinements.
