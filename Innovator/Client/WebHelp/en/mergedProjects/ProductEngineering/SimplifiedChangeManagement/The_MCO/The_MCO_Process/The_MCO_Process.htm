The MCO Process

The Manufacturing Change Order (MCO) process is used as a simplified change management process to edit an Approved Manufacturers List (the AML tab on Parts) after a Part has been released. This simplified process is not CMII compliant and modifies the part without change to a parts revision. A specialized user interface allows a user to edit the AML without allowing edits to other Part properties or relationships.

It will help to have the MCO form in front of us to refer to it, as we go through the required steps.

 

These are the first fields that need to be filled out, either by the creator of this MCO, or by the person entering the information for the creator.

Submitting an MCO

  1. From the TOC, select Change Management>MCOs, The MCOs menu appears.

  2. Click Create New MCO. A blank MCO form appears. following fields are read-only and server-assigned:

    • MCO Number - read only, assigned by the server.
    • Status - read only, shows the life cycle state of the MCO.

  1. Fill out the information for the following properties:

    • Status - read only, shows the life cycle state of the MCO.
    • Change Category – leave as server default, Phase In.
    • Originator – default value, the creator of this MCO.
    • Change Analyst – default value, person or group assigned to act as CSI in MCO workflow process.
    • Date Originated - default value, the date on which this engineering change notice was saved.
    • Release Date – automatic, the date on which this MCO is released.
    • Reason for Change – enter the description describing the change purpose of this MCO.
    • Change Description – enter the description of the change.

  2. Under the Attachments tab, specify the files that carry required information for this MCO.
  3. Under the Affected Parts tab, specify the Part for AML change.
  4. Save, Unlock, and Close the MCO.

Once you save the MCO, it goes to the In Basket of the Change Analyst, and the Prepare Change activity begins. The Change Analyst is responsible for the procedures described in the following sections.

Preparing to Change the MCO

  1. From TOC, go to My Innovator>My InBasket, and open the Prepare Changes activity.

  2. If the MCO does not meet criteria it may be voted to Cancel, but for this explanation the Change Analyst reviews the MCO and votes to Submit the change.

Once you submit the MCO, it goes to the In Basket of the Affected Parts Assigned Creator and the Draft Changes activity begins. The MCO promotes to the In-Work state and the Assigned Creator of the Affected Parts is responsible for the following steps.

Drafting Changes to the MCO

  1. From the TOC, select My Innovator>My InBasket, and open the Draft Changes activity.

  2. Select the Affected Part for the AML change. Right-Mouse-Click (RMB) the Affected Part row and select Launch AML Editor. The selection is available during the In-Work state.

 

The AML editor is a special dialog used to edit the AML of Parts that are released while preventing the editing of any other properties or relationships of the Part.

 

  1. Modify the AML List by adding, deleting, or modifying any relationship tab. After making the desired changes save the AML Editor.

  2. Update the MCO by voting to Complete the Drafting Changes activity.

Once the Drafting Changes activity is completed, it goes to the In Basket of the Change Analyst, and the Initial Review activity begins. The Change Analyst is responsible for the following procedure.

Initial Review of the MCO

  1. From the TOC, go to My Innovator>My InBasket, and open the Initial Review activity.

  2. Make certain that the parts AML List has correct information in it.

  3. Verify that all other information is correct and that all necessary files are attached.

  4. Update and vote to promote the MCO to Fast Track Approve, Full Review, or Reject.

At this point all the data necessary has been entered into the MCO. No matter which route the MCO takes - fast track, full review, or reject, there are three possible activity outcomes – Release Change, Review Changes, or back to Draft Changes for modification. Refer to The MCO to track the workflow process as the MCO completes each activity.
