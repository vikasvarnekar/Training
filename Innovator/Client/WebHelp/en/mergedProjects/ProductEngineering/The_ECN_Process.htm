The ECN Process

The Engineering Change Notice (ECN) process is used to add new parts or documents, to change parts or documents, or to mark them for deletion.  Let's take each one of these actions and go through its process step by step. Keep in mind that only the members of the CM group can initiate the ECN process. It will help to have the ECN form in front of us to refer to it, as we go through the required steps.

Taking an ECN action

  1. From the Navigation pane, select  Change Management>ECNs. The following menu appears:


  2. Click Create New ECN. A blank ECN form appears.

  3. Enter the appropriate information  in the following fields:

    • ECN Number - read only, server assigned sequence

    • Title - the title of the ECN

    • Status - read only, shows the life cycle state of the ECN

    • Basis - check either the Administrative or Physical Hierarchy

      • Physical hierarchy refers to the part itself and any document associated with it. A bug in a software would be classified in the physical hierarchy.

      • Administrative hierarchy refers to anything that is connected with the process of running the business. An error in the expense report procedure would be classified in the administrative hierarchy.

      • Priority - the priority of this change, the lower the number the higher the priority.

      • Assigned Creator - the engineer in charge of all technical aspects of this change notice, also referred to as the Owner.

      • Effectivity Date - the date on which this change notice goes into effect in production.

      • Customer Approval Required - check if the change requires customer approval.

      • Description - the description of the change.

      • Special Instructions - any special instructions that accompany this change.

    • On the ECRs tab, add any ECRs that are combined or included in this ECN.

    • On the Files tab, specify the files that carry required information for this ECN.

Adding a new Part or Document

  1. Click the Create Item icon  on the Affected Items tab to add an Item.
     

  2. Enter the following information:

    • Type - select Part or Document from the drop down box.
    • Old Number - for adding a new part, leave this blank.
    • Old Revision - read only field, the revision of the part.
    • Action - since we are adding a new part, select Add.
    • Interchangeable - leave it blank.
    • New Number - press F2, find and select the Part or Document that is being added.
    • New Revision - leave blank.
    • In Build- leave blank.
    • In Service - leave blank.

  3. Click  to save and unclaim the ECN.

Changing an Existing Part or Document

  1. Click the Create Item icon on the Affected Items tab to add an item.

  2. Enter the following information:

    • Action - select Change from the dropdown box.

    • Old Number - right-click in the cell and click Open. Select the Part or Document that is being changed.

    • Old Revision - automatically filled in.

    • Interchangeable - a change is defined as interchangeable, if the new part can be replaced or modified without causing any change in functionality to the surrounding parts. If the interchangeable flag is checked, the same part number will be retained.  Otherwise, a new part number will need to be assigned.

    • New Number - when the interchangeable flag is set, the superseding number is automatically filled in.  Otherwise, the CSI should enter the new part number.

    • New Revision - leave blank; the revisions will be entered automatically by Aras Innovator according to the Interchangeability rules.

    • In Build - select the action to take on the parts that are currently being built; the choices are: Use Existing, Rework, or Scrap

    • In Service - select the action to take on the parts that are currently in service; the choices are: Use Existing, Rework, or Scrap.

  3. Click  to save and unclaim the ECN.

 

An icon appears automatically in the Type column when you save the ECN.

Deleting an existing Part or Document

  1. Click the Create Item icon on the Affected Items tab to add a line to the grid.

  2. Enter the following information:

    • Type - select Part of Document from the drop down box.

    • Old Number - right-click in the cell and select Open. and select the Part or Document that is being deleted

    • Old Revision - filled in automatically with the Affected Number

    • Action - select Delete from the drop down box

    • Interchangeable - leave blank

    • New Number - leave blank

    • New Revision - leave blank

    • In Build - select the action to take on the parts that are currently being built; the choices are: Use Existing, Rework, or Scrap

    • In Service - select the action to take on the parts that are currently in service; the choices are: Use Existing, Rework, or Scrap

  3. Click  to save and unclaim the ECN.

This process does not actually delete the part or document from the BOM of its parent part. Delete serves as a tag, to indicate to the CSI that the BOM needs to be modified.  

Once the ECN is saved, the creator needs to submit it.

Submitting the ECN

  1. From the Navigation pane, select, My Innovator>My In Basket, and select the Submit ECN activity.
  2. Update the activity for this ECN, and vote to Submit.

Once you submit the ECN, it goes to the CSII for the ECN Planning activity. During this activity the CSII may provide the following information: the new part numbers for the parts that are not interchangeable and that are being changed; affected items that may be up the parent tree from the actual part being changed; affected documents of the parent assemblies (such as BOMs), etc. Since each company will have their own rules regarding new part numbers or new revision generation, the CSII is responsible for entering this information manually. The only automatic update that Aras Innovator will do, is if the part is being changed, and it is interchangeable, it will be advanced to a new revision. This automatic update will take place once the ECN is released.

Once the ECN Planning activity is done, the ECN follows the workflow process laid out in The ECN.
