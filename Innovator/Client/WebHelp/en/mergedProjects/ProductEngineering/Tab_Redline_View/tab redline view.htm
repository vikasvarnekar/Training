Tab Redline View

A Tab Redline View shows all changes made to relationships for all versionable ItemTypes. The graphical display is most useful when applied to Bills of Materials and Approved Manufacturer Lists using color-coded and strikethrough formatting. The tab view shows “redlines” against a previous revision displaying added, updated, and deleted items. Let's look at a Part BOM relationship before any changes.

Mandatory Redline Conditions

In most cases, the relationship grid will appear exactly as it does in previous versions. Redlines appear only if all of the following conditions are met:

  1. The relationship tab is in Redline mode. Click the  icon in the  relationship toolbar.

  2. The grid search bar is not visible. The grid search bar is disabled automatically when the redline icon is selected.

  3. The source item has a previously released generation.

  4. The relationships differ from the previously released generation

Entering Redline Mode

  1. From TOC, select Design, Parts, and open a Part with requirements 3 and 4 described above.
  2. In the BOM tab select the Redline View icon   in the relationship toolbar. Redline formatting of changes since the last revision appear.

Redline Format Adding / Deleting / Replacing

  1. A new Part BOM relationship is created with Sequence = 7 and Part Number = C-23456. This displays as blue text indicating the relationship was added.

  2. Row 6 (Part Number SM-120003) is deleted. Row 6 displays as red text with a full strike-through. It has been completely removed and the whole row is redlined.
