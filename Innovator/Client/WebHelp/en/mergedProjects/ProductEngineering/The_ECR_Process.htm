The ECR Process

The ECR process begins with the ECR form. Although there is just one form, different users are responsible for entering data in specific fields. To get a good overview of the ECR process, please see The ECR. The ECR process is described in more detail here.

Let's look at the ECR form.

 

Submitting an ECR

  1. From the Navigation pane, select Change Management>ECRs. The following menu appears:
     


  2. Click Create New ECR. A blank ECR form appears. The ECR Number is automatically assigned by the server when you save the form.

  3. Enter information in the following fields:

    1. Title - the title of the ECR should be as descriptive as possible.

    2. Administrative or Physical Hierarchy:

      1. Physical hierarchy refers to the part itself and any document associated with it. A bug in a piece of software would be classified as a physical hierarchy.

      2. Administrative hierarchy refers to anything that is connected with the process of running the business. An error in an expense report procedure would be classified as an administrative hierarchy.

    3. Corrective Action or Product Improvement - specify whether the ECR is a result of an already occurring problem that needs a corrective action, or whether this ECR is taking a proactive approach in order to prevent a possible problem, thereby improving the product.

    4. Requested By - the person who requests the change.

    5. Source - the source of the ECR - Internal, Customer, or Supplier.

    6. Proposed Solution - the solution proposed by the ECR Creator.

    7. Non-Recurring Cost Estimate - select the Less than $200 checkbox if the estimate is less than $200. Enter the full estimate amount in the field below the checkbox.

    8. Recurring Cost Estimate - enter the estimate amount and select either Up or Down from the dropdown menu.

    9. Tech Review - Select the Tech Review priority.

    10. Assigned Creator - Click the ellipses to access the Identity dialog box and select the appropriate Identity.

    11. Tech Review and Recommendation - Enter technical review information along with any recommendations in this field. Click the down arrow in the Problem Status field to select the appropriate status. Click the down arrow in the Solution field to select the appropriate solution.

  4. Click the Create Item icon  to add entries to the Affected Items tab as follows:

    • Action - Click the down arrow in the Action field to specify the appropriate action to take.

    • Old Number - Click the ellipses in the cell. The Change Controlled Item dialog box appears. Search for for the Part/Document/CAD Document that is to be changed through this ECR.

    • Old Revision - read only field.

    • Interchangeable - applicable when the action is change; indicates whether the new part will be interchangeable with the old part.

    • New Number - Click the ellipses in the cell to access the Change Controlled Item dialog box and select the appropriate item.

  2. From the Navigation pane, select My Innovator>My InBasket. The ECR should appear there, since you were the Creator of the ECR.

  3. Check over the data entered, and when the ECR is complete to the best of your ability, update the activity and vote Submit.

Once the ECR is submitted, it goes to the CSI's InBasket, and the Review ECR activity begins. The CSI is responsible for the following steps.

Reviewing the ECR

  1. From the Navigation pane, click My Innovator>My In Basket, and open the ECR. If the ECR does not meet criteria, you may wish to reject it immediately. To do that, update and vote to Reject.

  2. Verify that all information entered so far (see the Submit ECR activity above) is correct.

  3. Enter the following information:

    • Fast Track - check this box if this ECR will follow the Fast Track work flow.

    • Priority for Tech Review - the lower the number, the higher the priority.

    • Assigned Creator - assign the technical Owner of this ECR - the engineer who will be responsible for technical review of the problem and the ramifications as well as the review of the solution.

  4. If this ECR is in response to a reported problem, go to the PRs tab and enter all the filed PRs that should be resolved with this ECR.

  5. Verify that all items that are affected by this ECR are listed under the Affected Items tab.

  6. Verify that all the necessary files are attached under the Files tab.

  7. Save, Unlock, and Close the ECR.

  8. Update, and vote to send to Tech Review.

Once the ECR is sent to Technical Review, the Assigned Creator (also known as the Owner) of the ECR receives it in his In Basket. The Assigned Creator performs the technical review according to the company's policy and procedure, after which he is responsible for the following steps.

Performing the Technical Review of the ECR

  1. In the Navigation pane, select My Innovator>My InBasket, and open the Review ECR activity.

  2. Fill in the following properties:

    • Problem Status - select the appropriate value from the drop down box - Confirmed, Not Confirmed, or Another Problem.

    • Solution - select the appropriate value from the drop down box - Requester Solution or Another Solution.

    • Nonrecurring Cost Estimate - check the box if it's less than $200, and enter the actual estimate in the text box. This is the one-time cost estimate of implementing the solution for the ECR.

    • Recurring Cost Estimate - enter the recurring cost estimate in the text box. This is the difference in the unit cost of the part as a result of the change. Select UP or Down from the drop down box to indicate if the unit cost went up or down as a result of the ECR.

    • Key Implementation Timing Factors - enter the key timing factors for the implementation of the ECR.

  3. Verify that all information in the ECR is correct.

  4. Save, Unlock, and Close the ECR.

  5. Update, and vote to Complete.

Once the Assigned Creator is done with the technical review and has updated the activity, the ECR enters the Route ECR activity, assigned to the CSI. Here the CSI basically has to make a decision whether to send the ECR on the fast track approve, or whether the CRB involvement is necessary for approval.

Routing the ECR

  1. From the Navigation pane, select My Innovator>My In Basket, and open the ECR.

  2. Make certain that the Key Implementation Timing Factors field has the correct data in it.

  3. Select the Priority for implementation - the lower the number the higher the priority.

  4. Verify that all other information is correct and that all necessary files are attached.

  5. Click  to save and unclaim the ECR.

  6. Update and vote to either Fast Track Approve or CRB Approve.

At this point all the data necessary has been entered into the ECR. No matter which route the ECR takes - fast track or CRB, there are two final outcomes that are possible - ECR Approved or ECR Disapproved. Refer to The ECR to track the workflow process as the ECR completes each activity.
