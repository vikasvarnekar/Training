Configuring Permissions

This topic describes the out-of-the-box Permissions used in the Product Engineering (PE) application. In Aras Innovator, Item permissions define the access rights that identities have for performing actions on an item.

The following figure shows the Permissions accordion for an ItemType. The ItemType tab enables you to define permissions for this ItemType. Check the Is Default box to indicate that a particular Permission is the default for an Item.

Items may be associated with Life Cycle Maps. The Life Cycle Maps can override the default item permissions by setting a Permission for a particular item Life Cycle state as shown in the following figure.

The Access accordion tab of the Permission item view does not have the Add column. You can configure the create permission by adding the appropriate identity to the Can Add accordion tab for the relevant ItemType.

As an Administrator, you can change permission configurations using the standard procedures for granting access rights. When you grant the Can Change Access right, make sure that the granted individual is aware of and understands the responsibilities and requirements. Users can set Private permissions on individual items.
