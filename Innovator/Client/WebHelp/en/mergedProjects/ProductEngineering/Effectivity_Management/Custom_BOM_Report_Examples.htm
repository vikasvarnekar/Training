Custom BOM Report Examples

The following examples demonstrate how you can create custom BOM reports. 

Adding a New Column Using an Item Property

This example describes how to customize a multilevel BOM Report:

Use the following procedure to add the Unit column between the Quantity and AML Status columns to show the Unit property of the Part items:

  1. Search for the PE_multilevelReport Query definition and open it for editing.

  2. Click Show Editor on the PE_multilevelReport Query Definition sidebar.

  3. Click the Properties button on the RootPart row. The Properties dialog box appears.

  4. Add the Unit property to the Properties dialog box and click Save.

 

  1. Take steps 3-4 for the Part row representing the Part Query item.

Both Root and Part Query items should have the same properties in a Query definition. Otherwise, Aras Innovator returns an error. This requirement does not apply to other Query items associated with the Query definition.

  1. Click Save, Unclaim, and Close on the PE_multilevelReport Query definition toolbar.

  2. Search for Multilevel BOM Report and open its item view for editing.

  3. Go to the Stylesheet editor tab.

  4. Add the <th>Unit</th> table header cell element after <th>Quantity</th>.

  1. Add a table data cell element for the unit property values after the one for quantity.

       <td rowspan="{$rowcount}" width="120px">
                <xsl:value-of select="unit'/>
        </td>

 

  1. Click Apply on the Stylesheet editor tab.

  2. Click Done on the Multilevel BOM Report toolbar. The Unit column is added to the Multilevel BOM Report.
