Configuring BOM Reports

The following BOM Reports retrieve data using Query definitions:

  • BOM Costing Report - PE_costingReport

  • Multilevel BOM Report - PEmultilevelReport

  • BOM Quality Rollup Report - PE_quantityReport

To customize these reports you have to change their Query Definitions and Stylesheets. Select Administration>Reports from the TOC. 

By default, BOM reports are limited to 1000000 rows. Opening a larger BOM structure causes an error similar to the following:

If your company manages smaller BOM structures, you can limit BOM Reports to a specific number of rows using the following procedure:

  1. Search for the PE_ExecuteQdReportMethod method and open its item view for editing.

  2. Go to the PE_ExecuteQdReportMethod method editor.

  3. Change 1000000 to the specific number of all occurrences of the following if statement:

if (_partsTree.Count >= 1000000)
{
     throw new
     Aras.Server.Core.InnovatorServerException (errorLookup.Lookup ("PE_ReportIsVeryLarge"));
}

  1. Click  on the PE_ExecuteQdReportMethod method toolbar and close the item view.
