Activity and Assignment Profile

Different businesses use Activities, assignments and sub tasks differently. Multiple Assignments per Activity with measured work is the profile adopted in the standard solution but other profiles are supported. Determining what profile will be used is most important for all implementations both new and upgraded.

Activity Leader only

There are no sub Assignments, a single manager is specified for each Activity but there is no work estimate. Completion of  the Activity may be estimated and rolled up as a percentage of duration to parent WBS Elements. This profile is illustrated in Walkthrough1

Multiple Assignments

Each Activity can have zero or more assignments and each assignment has a work estimate. Full or partial completion of each assignment may be reported, and assignment completion data is rolled up to each activity and their parent WBS elements. This profile is illustrated with no work estimate in Walkthrough2 and with a work estimate in Walkthrough3. The benefits of collecting actual data using Time Reports is illustrated in Walkthrough4.  This profile supports the standard Release 8.0 and earlier case where there was a single assignee.

Other Use Cases

Other profiles can be implemented by customizing the update and activity completion rules to suit the business process

To implement an Activity and Assignments Profile

  1. Determine the business process for Activities and Assignments
  2. Determine the business process for Time Reporting
  3. Create Project Templates as required
  4. Customize the Rollup Calculation for Project Update as required
  5. Customize the Activity and Assignment Completion calculation as required
  6. Tailor the Activity Completion Worksheet as required
  7. Customize the Scheduling Calculation as required
  8. Test and validate the selected processes
