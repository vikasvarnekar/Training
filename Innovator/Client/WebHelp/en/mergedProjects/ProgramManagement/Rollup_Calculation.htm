Project Update

  1. The rules to be executed during Project Update  may be specified by name for each individual project
  2. See also Selecting when Project is Updated
  3. The standard rules, named 'individualUpdate', executes the following
    1. Confirm that the Project is ready for update
    2. Unlock all WBS and Activities in the Project
    3. Promote Activities from Pending to Active on their start date
    4. Calculate status of Activities, including status colors,  and rollup to WBS
  4. The default Project Update name is 'individual Update'
  5. The names of alternative Project Update rules will be the names of Innovator Methods which implement them.
  6. Alternate Project Update methods may in turn call an arbitrary sequence of Methods

To select Default Project Update Method

  1. Edit the Project ItemType
  2. Edit the update_method Property to select the default Method

To enable Alternate Project Update Method for individual Projects

  1. Implement custom behavior using Methods
