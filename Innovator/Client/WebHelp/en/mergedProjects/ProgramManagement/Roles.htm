Roles

Roles are used in Project Templates to designate what type of personnel would be responsible for a specific task. For example, a Design Engineer may be responsible for designing a DFMEA for a particular part. So, for that activity the responsible role may be Design. Later, when the Project is created from the template, the Project creator will fill in the specific design engineer who will be responsible for this task.

Roles are stored in the Project Role list. This list is then used as the data source for the Role property in the project template. The System Administrator may also need to see this list when setting up identities, in order to add a description field to each identity (see Setting Up Identities).

To create the Project Role list:

  1. From TOC, Administration, select Lists.
  2. Search for and open the Project Role list for edit.

 

  1. Click on the New Item icon and add entries as required.
  2. Save, Unlock, and Close the list.
