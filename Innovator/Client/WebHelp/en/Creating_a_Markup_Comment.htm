Creating a Snapshot Comment

Snapshot comments are comments that contain a snapshot of the file along with the text comment.

Note: Snapshot comments can be created with or without markups.

To create a snapshot comment, do the following:

  1. Click on any one of the viewers or its associated files in the sidebar of the item tear-off window. This displays the file in the view mode.
  2. Bring the file to the required view state by using the viewer toolbar. For more information on the toolbar options, refer to Viewer Toolbar.
  3. Switch to markup mode by clicking on the markup toolbar. This causes the existing view state to freeze. View functions are disabled and Markup functions are enabled.
  4. Ensure that the Include Image checkbox is turned on. Now the message created by the user will include the image that is currently displayed in the viewer.

  1. In the markup mode, create any markup notations (if desired) using the markup toolbar. For more information on the toolbar options, refer to Markup Toolbar.

  1. Enter a text comment (if desired) in the discussion toolbar.
  1. Click on Comment to save the markup comment.
  2. The new message which includes a thumbnail of the image with the markups is displayed in the discussion panel. The Include Image checkbox is removed and the viewer returns to view mode.

If the user decides not to save the markup notations,  then he can click on the View button at anytime to exit from the markup mode.

Thumbnail Image Display

The thumbnail of the markup image is displayed in the discussion panel along with the comment. This can be expanded and collapsed to view the markups more clearly.

Click on the "expand/collapse" icon in the upper left hand corner to expand or collapse the thumbnail in a snapshot style within the discussion panel.

Clicking on the thumbnail itself displays the markup image in the viewer.
