Creating a Relationship with a Related Item

A Relationship binds two Items - the source (parent) Item and the related (child) Item, making it possible for the source (parent) Item to reference the information held in the related Item.  The relationship is created by starting at the source (parent) Item, and creating a link to the related (child) Item.

In this Relationship type, it is mandatory to add a Related Item.  You can either add an existing Item or create a new Item.

Example:  Let us define a Part-BOM relationship with the Part ItemType as the source Item.  In our example, let us add related (child) Items such as Nuts, Cushion- Seat, Armrest, to the Chair Item (Part).

To create a Relationship with Related Item:

  1. Log on to Aras Innovator.
  2. From the TOC, navigate to the source (parent) item.
  3. Let us consider the Part ItemType as the source (parent) in this example. Select Design>Parts from the Navigation pane.
  4. From the main grid, select the source (parent) Item for which you want to add a relationship.

  1. In the BOM tab, click  (Select Items). The Search dialog appears:

  1. Select the source (parent) Item to add  to the source (parent) Item (Chair). A new row appears in the BOM tab.

You need to provide values for the related (child) Item properties. In our example, the Sequence property is auto-populated. The auto-population depends on the system property. We need to provide values for following properties:

Part Number

Revision

Name

Type

Quantity

Note: The Properties for each Item depend on the ItemType and are configured by the administrator.  Thus, the properties for each ItemType may differ, depending on the configuration done by the administrator.
