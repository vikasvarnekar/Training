CUI Controls

Location

Corresponding file: Innovator/Client/Modules/cui/cuiControls.js

Overview

CUI Controls (general term) - are special wrappers built to connect pure components and CUI controls definitions.

CUI Controls (module) - is a provider of metadata for CUI Layout. Based on the provided control type, CUI Controls module can choose the exact implementation of UI component. For now, the module provides information about the constructor, associated web-component name, init- and event handlers.

Information below relates to both of these concepts. Public API section relates to the module itself, while other sections provide infromation about general concept.

Public API

Common approach of working with CUI Controls module is the following:

import getControlMetadata from './cuiControls';
const controlMetadata = getControlMetadata(type); // e.g.: type === 'ToolbarControl';
// the value would be like:
// {
//  constructor: Function,
//  webComponentName: String,
//  ?initControl: Function,
//  ?eventHandler: Function
// }

Describing the purpose of each property:

  • .constructor - the method specifying the way control would be instantiated. In most cases constructor would create an HTML element/Web component using provided webComponentName (if any) and populate this DOM node with attributes like class, style and others.
  • webComponentName - holds the name of a web component to be used if applicable.
  • initControl - the method which defines how the control would be initialized. In most cases this method should contain the logic which subscribes cuiControl to events and creates the structure of nested controls
  • eventHandler - the method describing how control should react on the events which are handled by CUI Layout

This metadata would be collectioned by CUI Layout and used during layout's initialization.

How it works

CUI Controls operate in scope of CUI Layout - so all the input data is received from it as well as the outputs would be used by it.
