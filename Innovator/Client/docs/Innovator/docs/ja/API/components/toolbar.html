Toolbar Component

ロケーション

Innovator/Client/Modules/components/toolbar.js - ツールバーコンポーネントを含むメインファイルです。 パブリックAPIとコンポーネント定義はここに配置されます。

Innovator/Client/Modules/components/toolbarTemplates.js - ツールバー内のさまざまな要素をレンダリングするためのメソッドを含むファイルです。 templatesとformattersがあります：

  • Templatesは、ツールバー自体とネストされた要素（抽象アイテム）のレンダリングに必要なサービスメソッドです。
  • formattersは、特定のタイプのネストされたツールバー要素をレンダリングするために必要なパブリックメソッドです。 ボタン、画像、テキスト、セパレータなどのデフォルトのformattersがあります。

Innovator/Client/Modules/components/trimSeparators.js - 適切なセパレーターレンダリングに必要なメソッドを備えたユーティリティファイルです。 この方法により、近くまたは最初/最後の位置に冗長なセパレーターが残っている場合の解決が可能になります。

目的

ツールバーはWeb-componentであり、その目的はツールバーとツールバーのようなコンポーネントをレンダリングすることです。 デフォルトのコンポーネントが提供する機能では不十分な場合は、Toolbarを基本クラスとして使用して独自のクラスを作成できます。

ツールバーコンポーネントが現在使用されている場所の例：

  • メインヘッダー
  • タイトルとコマンドバー
  • ページ付け （継承クラス）

パブリックAPI

WebコンポーネントAPI:

  • ツールバーはWebコンポーネントの実装ですので、すべての機能をサポートしています。 以下の情報は、ツールバーに対して行われたカスタマイズに関する情報を提供します。 一般的な情報については、以下のリンクをご覧下さい。 MDN Web Components resource.

コンストラクタ:

  • document.createElement('aras-toolbar'): HTMLCustomElement - ツールバーの新しいインスタンスを作成します (そのうえまた<aras-toolbar></aras-toolbar>).

(HyperHTMLElementから)継承されたメソッドとプロパティが以下：

  • .connectedCallback(): void - role="toolbar"を設定し (HTML経由で提供されない場合), オーバーフロー動作を初期化し、イベントリスナーを初期化します。

  • .disconnectedCallback(): void - イベントサブスクリプションを削除します。

  • .attributeChangedCallback(attrName: string): void - roleとoverflowの変化を観察し、 変更が発生すると、更新されたツールバーをレンダリングします。

  • .booleanAttributes [静的な, リードオンリー] - デフォルトの属性はオーバーフローと拡大されたです。ということは、値を指定する必要はありません-それらの存在自体がツールバーの動作を定義します。 このプロパティはHyperHTMLライブラリによって提供され、ここで提供される値は* .observedAttributes *で提供される値と連結され、変更を監視する完全なリストを形成します。

  • .observedAttributes [静的な, リードオンリー] - 変更のために観察される属性です。 デフォルトでは、値は roleです。さらに、.booleanAttributesの値が監視対象のリストに追加されます。 属性が変更されると、.attributeChangedCallbackが起動します。

メソッドとプロパティ:

  • .data:

    • set Map(<key: string, 値: object>) - ツールバーデータのセッター。Mapを受け入れます。keyはネストされた要素の識別子で、 valueはこの要素を記述するオブジェクトです。 データが設定されると、要素がコンテナに配置され、ツールバーが自動的に再レンダリングされます。
    • get Map(<key: string, 値: object>) - ツールバー要素を含むマップを返します。

  • .container:

    • set string[] - これは、ツールバーの「左」コンテナのセッターです。 文字列キーの配列を受け入れます（dataで使用されるものと同じ）。 データがツールバーに設定されると、それらをツールバー内の 左側または右側に配置できます。 コンテナに入れられたアイテムは"左"とみなされます。 ほとんどの場合、値を変更する必要はありません。ツールバー要素は、要素の.rightプロパティに 基づいたdata割り当て中に自動的に配置されるためです。
    • get string[] - "左"のコンテナのゲッターです。

  • .rightContainer:

    • set string[] - containerと同じですが、右側のアイテム用です。
    • get string[] - containerと同じですが、右側のアイテム用です。

  • .on(eventType: string, callback: Function): Function - イベントをサブスクライブできるメソッドです。 イベントが発生するたびに、コールバックはイベントが発生した要素に関する情報で起動します。 要素がない場合（たとえば、ツールバーのパディング領域またはボタン間をクリックした場合）、または要素が無効になっている場合、 コールバックは発生しません。 コールバックの署名: (itemId: string, event: Event) => void | Promise<void> このメソッドはdetach関数を返し、作成されたサブスクリプションを削除できます。

  • .render(void): Promise - ツールバーのレンダリングのためのメソッドです。 このメソッドは、現在の状態とデータに従ってツールバーをレンダリングします。 ほとんどの場合、すべてのデータ変更により自動再レンダリングが行われるため、このメソッドを明示的に呼び出す必要はありません。 このメソッドは非同期に動作するため、DOMのツールバーの外観に依存している場合は、返されるPromiseの待機を検討してください。

  • static .extendFormatters(Object) - ツールバーに追加のformattersを提供できます。 オブジェクトを受け入れます。 オブジェクトのプロパティの名前は、要素のタイプの識別子として使用されます。 対応する関数はformattersとして使用されます。 formatterには次のシグネチャがあります: (item, options) => VirtualNode.

ツールバーの内部構造へのアクセスを提供するプロパティが以下:

  • .datastore - dataマップへの直接アクセスを提供するプロパティです。 このプロパティは、自動レンダリングを意味しないため、派生クラスでの使用に役立ちます。 注：このプロパティは、外部コードからの使用には推奨されません。

  • .templates - ツールバー自体のレンダリングとその要素のレンダリングの開始を担当する

    • defaultTemplates *への直接アクセスを提供するプロパティです。 このプロパティは、派生クラスでの使用に役立ちます。 注：このプロパティは、外部コードからの使用には推奨されません。

  • .focusedItemId - ツールバー内で現在フォーカスされているアイテムのキーへのアクセスを提供するプロパティです。 このプロパティは、派生クラスで使用してa11y関連の機能を使用する場合に便利です。 注：このプロパティは、外部コードからの使用には推奨されません。
