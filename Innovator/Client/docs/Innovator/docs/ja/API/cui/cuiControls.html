CUI Controls

ロケーション

対応するファイル: Innovator/Client/Modules/cui/cuiControls.js

概要

CUI Controls (一般用語) - というのは純粋なコンポーネントとCUIコントロール定義を接続するために構築された特別なラッパーのことです。

CUI Controls (モジュール) というのはCUI Layoutのメタデータのプロバイダーのことです。 提供されたコントロールタイプに基づいて、CUI ControlsモジュールはUIコンポーネントの正確な実装を選択できます。 現在、このモジュールは、コンストラクター、関連するWebコンポーネント名、初期ハンドラー及びイベントハンドラーに関する情報を提供します。

以下の情報は、これらの両方の概念に関連しています。 パブリックAPIセクションはモジュール自体に関連しています。一方、他のセクションは一般的な概念に関する情報を提供しています。

パブリックAPI

CUI Controlsモジュールを使用する一般的なアプローチは次のとおりです:

import getControlMetadata from './cuiControls';
const controlMetadata = getControlMetadata(type); // e.g.: type === 'ToolbarControl';
// the value would be like:
// {
//  constructor: Function,
//  webComponentName: String,
//  ?initControl: Function,
//  ?eventHandler: Function
// }

各プロパティの目的を説明しています:

  • .constructor - というのは、コントロールがインスタンス化される方法を指定するメソッドのことです。 ほとんどの場合、constructorは、指定されたwebComponentName（存在する場合）を使用してHTML要素/ Webコンポーネントを作成し、 このDOMノードにclass, styleなどの属性を設定します。
  • webComponentName - 該当する場合に使用されるWebコンポーネントの名前を保持しています。
  • initControl - というのは、コントロールの初期化方法を定義するメソッドのことです。 ほとんどの場合、このメソッドにはcuiControlをイベントにサブスクライブし、ネストされたコントロールの構造を作成するロジックが含まれる必要があります
  • eventHandler - というのは、CUIレイアウトによって処理されるイベントにコントロールがどのように反応するかを記述するメソッドのことです。

このメタデータはCUI Layoutによって収集され、レイアウトの初期化中に使用されます。

使い方

CUIコントロールはCUI Layoutのスコープで動作します- したがって、すべての入力データもCUI Layoutのスコープから受信されれば、出力もスコープで使用されます。
