CUI ContextMenu

ロケーション

対応するファイル: Innovator/Client/Modules/cui/cuiContextMenu.js

目的

CUIコンテキストメニューモジュールは、他のCUIコントロールと同じように機能し、 提供されたデータに従って　Context Menu を確立します。

パブリックAPI

エクスポートされた関数には次のシグネチャがあります:

function cuiContextMenu(control: ContextMenu, location: String, ?options: Object): Promise(controlDestructor)

その関数から返されたcontrolDestructorは次のメソッドを含む対象のことです:

  • show(): void methodによってメニューを開くことができます。 ただし、メニューが表示される前に、表示されるメニューエントリが計算されます。 メニューに表示項目がない場合、メソッドは戻ります。
  • destroy(): コンテキストメニューが不要になった場合は、void methodが必要になります。

使い方

他のコントロールの範囲でCUIコンテキストメニューがどのように機能するかをよりよく理解するには、CUI LayoutおよびCUI Controlsページをご覧ください。

なお、CUI ControlがCUI Layoutによって作成され、初期化されることは一般的ですが、CUIコンテキストメニューはそうではありません。 そのため、サーバー/スタブから受け取ったレイアウト構造を記述するCUIControls定義の一部には、CUIコンテキストメニューに関する情報が含まれていません。

cuiContextMenuに関する重要な注意事項が次のとおりである:

  1. サーバー/スタブから受信したデータには、ネストされたアイテムの単一レベルが含まれています。

    ネストされたアイテムに関する情報は、2つのセマンティックグループに分割されています: data 及び roots. dataプロパティはネストされたアイテムに関する情報を保持します。 一方rootsプロパティは、現在のレベルの下の最初のレベルに表示されるアイテムのIDのみを指します。

    これらのプロパティはオプショナルであり、メニューエントリが「親メニュー」であり、いくつかのネストされたエントリがある場合にのみ表示されます。

    したがって、通常のメニュー項目は次のようになります:

    "@id": "83C93103439243ADB2A8472FC743EB1B",

    "id": "83C93103439243ADB2A8472FC743EB1B",
    "label": "Open",
    "on_click_handler@keyed_name": "cui_default_mwt_onViewCommand",
    "on_click_handler@type": "Method",
    "on_click_handler": "74217FBDC2554604A543BDEFA8A64ED5",
    "on_init_handler@keyed_name": "initPopupItemInItemsGrid",
    "on_init_handler@type": "Method",
    "on_init_handler": "2B56E062A54245B7851FA09CFD224882",
    "name": "com.aras.innovator.cui_default.pmig_View",
    "itemtype": "B5175846B27145EEA5653DA35ED78BE4",
    "sort_order": "128",

    "section": "89E6EFE42BA04A3680CD125918BB82DF",

    ネストされた項目を持っているメニューエントリは次のようになります:

    "@type": "CommandBarMenu",

    "item_classification": "",
    // specifies ids of nested items
    "roots": [
        "com.aras.innovator.cui_default.pmig_edit" // refers to the "name" of nested item

    // nested items themselves

        {

            "parent_menu@type": "CommandBarMenu",
            "parent_menu": "45ACD568CFC0489BA7D5365C19208ADB",
            "name": "com.aras.innovator.cui_default.pmig_edit",
            "itemtype": "B5175846B27145EEA5653DA35ED78BE4",
            "sort_order": "2",
            "section@keyed_name": "com.aras.innovator.cui_default.popup_menu_for_item_grid",
            "section": "89E6EFE42BA04A3680CD125918BB82DF",
            "item_classification": ""
        }
    ]
}

  2. すべてのメニューエントリは、提供された on_init_handlerを使用して再帰的に初期化されます。

  3. メニューエントリ定義のすべてのimageプロパティはiconに置き換えられます。 このステップは、画像ローダーとの統合されたインターフェースを維持するために必要です。

  4. すべてのメニューエントリは、ターゲットコントロールに渡されるデータ構造の最初のレベルに移動します。 再帰なしでアイテムアクセスの単純さを保つために必要です。

  5. アイテムのrootsプロパティはchildrenに置き換えられます（すべてのデータが保存されます）

  6. アイテムの dataプロパティは削除されます（ネストされたアイテムは構造の最初のレベルに保存されるため）。

  7. すべてのメニューエントリが初期化されると、それらに必要なアイコンは SvgManagerを介して自動的にロードされます。

初期化の最後の手順は、コンテキストメニューコントロールにデータを渡し、パブリックAPIを返すことです。 データは、コントロールのrootおよびdataプロパティとして保存されます。
